The Problem
Coordinating a family is messy
01

Scattered communication

Shopping and chores are coordinated over WhatsApp messages and paper lists — nothing is shared or structured.

02

Duplicated & forgotten

Two people buy the same thing, or an item nobody claimed is forgotten entirely. No single live list.

03

No accountability

Who added what? Who bought it? There is no record of who did which task and when.

04

No spend awareness

Families have no shared view of how much a shopping trip costs or whether they are over budget.

FAKERNI02 / 12
The Solution
One shared, live source of truth

Households & Fakras. A family shares smart lists (“Fakras”) of things to buy or do — personal, group, or shared with specific people.

Real-time sync. Add or complete an item and everyone sees it instantly on web and mobile — no refresh, no double-buying.

AI assistance. Photograph a receipt or type “milk, eggs, 2kg rice” — Gemini fills the structured list for you.

Budgets & analytics. Each item has a price; the app tracks spend, alerts on overruns, and charts spending.

Accountability. An activity log records who added and completed every item, with timestamps.

Trilingual & mobile-first. Arabic (RTL), French, English — with push notifications and biometric lock.

Data Model
Core entities & relations

User → belongs to Households via Membership (role: owner / admin / member).

Fakra — a list. Its household link is optional, so a Fakra can be personal.

Item — belongs to a Fakra: name, quantity, unit, category, price, status, done-by.

ItemAttachment (photo), ActivityLog (who did what), FakraAccess (explicit share).

Three ways to access a Fakra

1 — You created it (personal)

2 — It belongs to your household

3 — It was shared with you

A single helper — require_fakra_access() — enforces all three, on every REST route and every WebSocket connection.

Technology & justification

Django + DRF — mature ORM, auth, serializers, throttling; build a correct, secure API fast.

Clients
React (web) + Expo/React Native (mobile) — shared logic; one JS codebase for iOS + Android.

Highlight · AI
Removing the typing with Gemini

Smart Add

“milk, eggs and 2kg rice” → structured items with quantity, unit & category.

Smart Scan

Photograph a receipt → items and prices extracted automatically.

Suggestions

Context-aware ideas for what else the list probably needs.

Smart Command

“mark milk done, remove the bread” → actions applied to the real list.

Reliable by design: every call passes a Pydantic output schema, so Gemini returns guaranteed-valid JSON — never fragile free text. If the key is missing or the API fails, it raises a clean error and the rest of the app keeps working (graceful degradation).

FAKERNI08 / 12
Highlight · Real-Time
Authenticated, spoof-proof channels
01

Household channel

ws/households/<id>/
Broadcasts family-wide events to all members.

02

Per-Fakra channel

ws/fakras/<id>/
Works for personal & shared lists, not just household ones.

03

Per-user channel

ws/notifications/
Joined from the JWT identity — can’t be spoofed. Live Dashboard.

The hard part: JWT auth doesn’t apply to WebSockets automatically. A custom ASGI middleware validates the token, attaches the user to the connection, and each consumer rejects connections the user isn’t authorised for.

FAKERNI09 / 12
Security
Defence in depth

Passwords — Django PBKDF2 hashing, salted, never stored in plaintext.

Auth — short-lived JWT access (60 min) + refresh (7 days) with silent renewal.

Brute force — rate limiting on login / register / reset → HTTP 429.

Password reset — single-use email OTP, 10-min validity, no user-enumeration leak.

Encryption at rest — OTP codes & device tokens encrypted with Fernet.

Authorization — centralised access checks on every endpoint & socket.

Transport — HTTPS redirect, secure cookies, HSTS when DEBUG=False.

Device — optional biometric app-lock (Face ID / fingerprint) on mobile.
